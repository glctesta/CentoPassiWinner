Panoramica del Sistema

CentopassiWinner è un pianificatore intelligente di percorsi progettato specificamente per la gara CENTOPASSI® 2026 (29 maggio – 1 giugno 2026). L'obiettivo della gara è attraversare esattamente 100 passi di montagna in auto nel minor tempo possibile, massimizzando il punteggio.

📂
Import Waypoints
Carica il file ufficiale GPX o XLSX con tutti i passi di montagna disponibili. Supporto drag & drop.
🧠
Ottimizzazione Automatica
Algoritmo greedy + 2-opt che seleziona i 100 WP ottimali e li distribuisce sui 4 giorni rispettando tutti i vincoli.
🛣️
Routing Professionale
Calcolo percorsi su strade reali via OSRM (open source) o GraphHopper. Tracciato GPX completo che segue le strade.
🤖
AI Day Optimizer
Claude AI (Anthropic) riordina i WP di ogni giorno per il percorso più efficiente, rispettando le strade vietate.
✏️
Editor Interattivo
Clicca sulla mappa per aggiungere o rimuovere WP giorno per giorno. Modifica manuale in tempo reale.
📥
Export GPX Multi-giorno
Scarica 4 file GPX separati (uno per giorno) o un unico ZIP. Naming convention con numero WP, giorno e posizione.

Il programma implementa tutte le regole ufficiali del regolamento. Ecco i vincoli principali gestiti automaticamente.

Vincoli Percorso

  • 100 passi esatti — né di più, né di meno
  • Totale ≥ 1600 km — distanza minima di percorrenza
  • Partenza >15 km dal primo passo (art. 6.2)
  • Partenza ≤450 km (aria) dal traguardo (art. 6.2)
  • Bonus 100°: il 100° passo deve essere entro 50 km dal traguardo
  • Bridge inter-giorno: il 1° WP del giorno N+1 deve trovarsi a 20–40 km su strada dall'ultimo WP del giorno N

Strade Vietate (Art. Regolamento)

⛔ Esclusione dalla classifica in caso di transito su: Autostrade · Tangenziali · Scorrimenti veloci (SSV) · Strade di Grande Comunicazione (SGC) · Circonvallazioni · Ring road / Highway / Motorway

Il programma verifica automaticamente la compliance tramite ispezione dei nomi strade OSRM e avvisa in caso di violazioni rilevate.

Waypoint Opzionali

Dopo l'ottimizzazione vengono selezionati 10 WP di riserva distribuiti proporzionalmente: più opzioni verso gli ultimi giorni (formula triangolare: G1=1, G2=2, G3=3, G4=4).

Guida all'Uso — Passo per Passo

1
Carica i Waypoints

Clicca sulla zona "📂 Carica Waypoints" oppure trascina il file. Formati supportati:

  • GPX — File ufficiale CENTOPASSI (es. Generale 2026.GPX)
  • XLSX / XLS — Foglio Excel con colonne: nome, latitudine, longitudine, quota, tipo

Dopo il caricamento, tutti i passi vengono visualizzati sulla mappa come marker colorati.

2
Posiziona il Traguardo

Clicca il bottone "📍 Posiziona Traguardo", poi clicca sulla mappa nel punto di arrivo (es. piazza della città di arrivo). Il traguardo è necessario per il calcolo del bonus 100° passo.

3
Avvia l'Ottimizzazione

Clicca "🚀 Ottimizza Percorso". L'algoritmo:

  • Seleziona i 100 WP con il punteggio migliore (quota, GP, distanza)
  • Li distribuisce sui 4 giorni rispettando km massimi e bridge 20–40 km
  • Applica nearest-neighbor + 2-opt per minimizzare la distanza totale
  • Sceglie 10 WP opzionali come riserva

L'operazione dura 10–60 secondi in base al numero di waypoint.

4
Calcola il Routing Stradale

Nella sezione "🛣️ Routing Stradale" clicca "Calcola Percorsi Reali". Il sistema interroga OSRM per ogni segmento WP→WP e costruisce il tracciato GPX che segue le strade reali. Questo passaggio può richiedere 2–10 minuti in base al numero di segmenti.

5
Ottimizza con AI (opzionale)

Nell'editor, seleziona un giorno e clicca "🤖 Ottimizza Giorno con AI". Claude AI analizza i WP del giorno e suggerisce l'ordine ottimale evitando backtracking e strade vietate.

6
Modifica Manualmente (Editor)

Nella sezione "✏️ Editor Percorso": seleziona un giorno → clicca WP verdi sulla mappa per aggiungerli → clicca WP colorati per rimuoverli. Le statistiche si aggiornano in tempo reale.

7
Esporta i file GPX

Clicca "📥 Scarica GPX Giorno N" per il singolo giorno oppure "📦 Scarica Tutti" per ottenere uno ZIP con 4 file GPX separati. I file sono compatibili con Garmin, Wahoo, komoot, Wikiloc.

Sfondo Mappa

Clicca il bottone 🗺️ in alto a sinistra sulla mappa per scegliere tra 8 temi: Standard, Satellite, Terreno, OpenTopoMap, Cicloviario, Scuro, Nautical, Acquarello.

Routing Stradale Professionale

Il sistema usa routing reale su strade (non linea retta) per produrre tracciati GPX accurati e verificare la compliance con le regole di gara.

Provider Chain

PrioritàProviderVantaggiLimite
1° GraphHopper API Avoid motorway+toll nativi, nomi strade italiani precisi Richiede API key, 500 req/giorno (piano free)
2° OSRM (demo server) Open source, gratuito, geometria overview=full No exclude nativo, compliance post-hoc
3° Haversine (fallback) Sempre disponibile, istantaneo Linea retta ×1.35, nessun GPX reale

Compliance Strade Vietate

OSRM non supporta il parametro exclude=motorway sul server demo. Il sistema usa invece ispezione post-hoc:

  • OSRM restituisce i nomi di ogni strada nel percorso (steps=true)
  • Il sistema confronta ogni nome con la lista FORBIDDEN_ROAD_KEYWORDS
  • Se trova violazioni, tenta un rerouting alternativo con waypoint intermedio
  • Le violazioni residue vengono segnalate nel pannello compliance

Parametri OSRM Usati

GET /route/v1/car/{lon1},{lat1};{lon2},{lat2} ?overview=full ← geometria completa (non semplificata) &geometries=geojson ← formato coordinate &steps=true ← nomi strade per compliance check &annotations=false ← disabilita dati extra inutili

Cache Automatica

Ogni percorso calcolato viene salvato in route_cache.json. Al successivo avvio il server ricarica la cache, evitando chiamate API ridondanti. La cache viene salvata ogni 50 nuove richieste.

Qualità Routing

Dopo il calcolo, la barra di qualità mostra:

  • % segmenti con routing reale (verde >80%, arancione 50–80%, rosso <50%)
  • Punti GPX totali — un buon routing ha >10.000 trkpt per giorno
  • Violazioni strade — lista delle strade potenzialmente vietate rilevate

AI Day Optimizer — Claude AI

L'ottimizzatore AI usa Claude Sonnet (Anthropic) per trovare l'ordine ideale dei waypoint di ogni singolo giorno, considerando tutti i vincoli della gara.

Come Funziona

  1. Estrae la lista dei WP del giorno con coordinate, quota e tipo (GP, sterrato)
  2. Calcola la matrice delle distanze haversine tra tutti i WP
  3. Invia il tutto a Claude con le regole complete della gara
  4. Claude restituisce un JSON con l'ordine ottimale e la motivazione
  5. Il risultato viene raffinato con l'algoritmo 2-opt locale
  6. Il vincolo bridge viene applicato forzatamente sull'ordine finale

Vincoli Comunicati all'AI

  • Finestra temporale giornaliera e km massimi
  • Vincolo bridge 20–40 km dal giorno precedente
  • Strade vietate: autostrade, tangenziali, SSV, SGC, circonvallazioni
  • Preferenza per strade secondarie (SP, SS, strade comunali)
  • Strategia altimetrica (salite al mattino, discese la sera)
  • Priorità ai Golden Point (3× punti) e gestione sterrati

Fallback Automatico

Se Claude non è disponibile (API key mancante o errore di rete), il sistema usa automaticamente Greedy Nearest-Neighbor + 2-opt come algoritmo di fallback — comunque molto efficiente.

Editor Interattivo del Percorso

L'editor permette di modificare manualmente il percorso giorno per giorno direttamente sulla mappa, dopo l'ottimizzazione automatica.

Accesso all'Editor

Dopo l'ottimizzazione, scorrere nella sidebar fino alla sezione "✏️ Editor Percorso". Selezionare il giorno con i tab G1–G4.

Aggiungere un Waypoint

  • I WP non ancora assegnati appaiono sulla mappa come cerchi verdi semitrasparenti
  • Cliccare su un WP verde → popup con bottone "+ Aggiungi al Giorno N"
  • Il WP viene inserito nella posizione ottimale nella sequenza del giorno
  • Le statistiche (km, ore) si aggiornano immediatamente

Rimuovere un Waypoint

  • I WP del giorno selezionato appaiono come cerchi colorati grandi con numero di sequenza
  • Cliccare su un WP nel percorso → popup con bottone "✕ Rimuovi dal Giorno N"
  • Il WP rimosso torna disponibile (cerchio verde) per altri giorni

Colori per Giorno

G1 — Viola G2 — Ciano G3 — Arancio G4 — Rosso GP — Oro ⭐ Sterrato 🟠

Inserimento Ottimale

Quando si aggiunge un WP, il sistema usa l'algoritmo "cheapest insertion": trova la posizione nella sequenza esistente che minimizza l'aumento di distanza, evitando backtracking.

Waypoint Opzionali

I 10 WP opzionali (riserve) sono già distribuiti per giorno nella sidebar dell'editor. Possono essere aggiunti al percorso con un click se serve sostituire un WP problematico.

Esportazione GPX

I file GPX prodotti sono nel formato standard GPX 1.1 e contengono sia i waypoint che il tracciato completo su strade reali.

Naming Convention Waypoint

Esempio: WP042 G2 07/25 │ │ │ └── Totale WP nel giorno │ │ └───── Posizione nel giorno (07°) │ └───────── Giorno di gara (Giorno 2) └─────────────── Numero WP ufficiale (042)

Naming File GPX

centopassi_2026_G1.gpx ← Giorno 1 centopassi_2026_G2.gpx ← Giorno 2 centopassi_2026_G3.gpx ← Giorno 3 centopassi_2026_G4.gpx ← Giorno 4 centopassi_2026_tutti.zip ← ZIP con i 4 file separati

Contenuto di ogni file GPX

  • <wpt> — Ogni waypoint con nome, descrizione, coordinate, quota
  • <trk> — Traccia completa su strade reali (se routing calcolato) o linea retta
  • <trkseg> — Un segmento per ogni tappa WP→WP
  • Migliaia di <trkpt> che seguono esattamente le strade

Configurazione e Setup

Requisiti di Sistema

  • Python 3.10 o superiore
  • Connessione internet (per OSRM, Claude API)
  • Browser moderno (Chrome, Firefox, Edge, Safari)

pip install -r requirements.txt # Dipendenze principali: # flask>=3.0 # requests>=2.31 # anthropic>=0.39 # python-dotenv>=1.0 # openpyxl>=3.1
